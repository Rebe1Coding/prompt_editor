# Prompt Editor

Бэкенд веб-приложения для редактирования промптов. Пользователь отправляет промпт —
сервис возвращает исправленную версию: без ошибок, двусмысленностей и логических противоречий.
Результат можно перегенерировать или скорректировать уточнением, финальная версия
сохраняется в MD-файл.

## Стек

- Python 3.12, FastAPI
- LangChain + OpenAI-совместимый клиент, провайдер — RouterAI
- SQLite — история сессий и правок
- uv — окружение и зависимости

## Установка

1. Установить зависимости:

   ```bash
   uv sync
   ```

2. Создать конфиг из шаблона и вписать ключ RouterAI в `provider.api_key`:

   ```bash
   cp config.example.yaml config.yaml
   ```

## Запуск

```bash
uv run python -m app.main
```

Сервер поднимется на `http://127.0.0.1:1488`. Swagger-документация — на `/docs`.

## Использование

1. `POST /api/prompts` с телом `{"prompt": "..."}` — отправить промпт. Ответ приходит
   потоком (SSE): сначала событие с `session_id`, затем токены по мере генерации.
2. `POST /api/prompts/{id}/regenerate` — новый вариант потоком, если текущий не понравился.
3. `POST /api/prompts/{id}/refine` с телом `{"instruction": "..."}` — точечная правка потоком.
4. `POST /api/prompts/{id}/finalize` — сохранить финальную версию в `data/results/*.md`.

Три генерирующих ручки отдают результат потоком токенов (Server-Sent Events), чтобы
текст был виден сразу, а не после полного ответа. Финализация и чтение — обычный JSON.
Полное описание ручек, формат потока и коды ошибок — в `docs/API.md`.

## Конфигурация

Все настройки — в `config.yaml`:

| Секция | Параметры |
|---|---|
| `provider` | ключ API, `base_url`, модель, температура |
| `storage` | пути к базе, папке результатов и системному промпту |
| `server` | хост и порт |

Системный промпт редактора лежит в `prompts/system_prompt.md` и перечитывается
на каждый запрос — править можно без перезапуска сервера.

## Структура проекта

```
app/
  config.py    # чтение config.yaml
  db.py        # SQLite: сессии и ревизии
  llm.py       # цепочка LangChain, сборка истории диалога
  storage.py   # сохранение финальных MD-файлов
  schemas.py   # модели тел запросов
  main.py      # FastAPI-приложение и ручки
prompts/
  system_prompt.md
docs/
  API.md
config.example.yaml
```

Данные (`data/`): база `prompt_editor.db` и финальные промпты в `data/results/` —
создаются автоматически при первом запуске, в репозиторий не попадают.
